# MVP

**Purpose**: Defines the minimum viable product scope, phased delivery, acceptance criteria, timeline, and success metrics.

**For Whom**: Agents managing development, tracking progress, prioritizing features.

## Overview

The MVP is phased across three releases. Phase 1 validates the core logging loop. Phase 2 adds the AI assistant. Phase 3 activates the publishing and growth loop.

## Key Sections

- [scope.md](./scope.md): MVP features broken down by phase
- [acceptance_criteria.md](./acceptance_criteria.md): Acceptance criteria per feature
- [timeline.md](./timeline.md): Development timeline and milestones
- [success_metrics.md](./success_metrics.md): MVP success metrics and KPIs

## Quick Reference

- **Phase 1**: Voice logging + build timeline
- **Phase 2**: AI assistant
- **Phase 3**: Publishing + watermark + discovery
