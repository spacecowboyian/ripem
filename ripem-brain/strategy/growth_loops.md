# Growth Loops

## The Core Viral Loop

```
User logs build sessions
       ↓
AI suggests publishing a build update
       ↓
User publishes (auto-watermarked content)
       ↓
Content shared on TikTok / Instagram / YouTube Shorts
       ↓
Viewers see RiPeM watermark
       ↓
Viewers download RiPeM
       ↓
New users log their builds
       ↓ (loop repeats)
```

## Loop Analysis

| Stage | Friction Level | Optimization Lever |
|-------|---------------|-------------------|
| Log sessions | Low (voice-first) | Reduce session logging to < 60 seconds |
| Publish suggestion | Low (AI-triggered) | Smart timing: suggest after milestones |
| Publish action | Low (one tap) | AI auto-generates content |
| Watermark visibility | High (depends on content quality) | Build quality content examples |
| App discovery | Medium (App Store friction) | Deep link from watermark |
| Download + register | Medium | Streamlined onboarding |
| First session logged | High (critical moment) | Onboarding push within first 24h |

## Secondary Growth Loops

### Community Discovery Loop
1. User discovers an interesting build in the feed
2. User follows the builder
3. Builder gets a notification ("Someone followed your build")
4. Builder logs more sessions to maintain audience
5. Audience engagement grows → more builders stay active

### AI Value Loop
1. User logs more sessions → AI gets more context
2. More context → better, more personalized AI answers
3. Better AI answers → higher retention
4. Higher retention → more logged sessions

## Viral Coefficient Target

**K-factor target**: ≥ 0.3 (30 new users per 100 active users per month from watermark)

A K-factor of 0.3 combined with paid/organic acquisition creates sustainable growth.
